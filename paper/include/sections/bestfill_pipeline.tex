\subsection{Problem Statement}
\label{sec:bestfill_problem}

Given a context $\operatorname{GF}(257)$ rotation $\mathbf{q} = (a_q, b_q)$
and a graph of $N$ nodes each carrying their own rotation state
$\{(a_i, b_i)\}_{i=1}^{N}$, the resolver returns:

\begin{equation}
  i^* = \operatorname*{arg\,min}_{1 \leq i \leq N}\;
        d^2(\mathbf{q}, \mathbf{n}_i)
  \label{eq:resolver_argmin}
\end{equation}

where $d^2$ is the squared Euclidean distance in the Galois field coordinate
space.  This is the system's sole inference primitive---all generation,
retrieval, and reasoning reduce to repeated resolver calls.

\subsection{GF(257) Geometric Distance}
\label{sec:gf_distance}

Each node in the thermodynamic field carries a composable affine state
$(a, b) \in \operatorname{GF}(257)^2$ where:

\begin{equation}
  f(x) = (a \cdot x + b) \bmod 257, \qquad a \in \{1, \ldots, 256\},\; b \in \{0, \ldots, 256\}
  \label{eq:gf_affine}
\end{equation}

The resolver computes the distance between a query context and every candidate
node as:

\begin{equation}
  d^2(\mathbf{q}, \mathbf{n}_i) = (a_i - a_q)^2 + (b_i - b_q)^2
  \label{eq:gf_distance}
\end{equation}

This is pure integer arithmetic---no floating-point operations, no popcount,
no matrix multiplies.  The geometric interpretation is direct: $a$ encodes
the \emph{multiplicative} (scaling/rotation) identity, $b$ encodes the
\emph{additive} (translational) offset.  Nearby nodes in
$\operatorname{GF}(257)$ space share similar rotational histories.

\subsection{Single-Pass Parallel Scan}
\label{sec:single_pass}

The resolver executes as a single-pass parallel scan across all $N$ nodes:

\begin{enumerate}[nosep]
  \item Each GPU thread evaluates $d^2(\mathbf{q}, \mathbf{n}_i)$ for its
        assigned node (2 subtractions, 2 multiplications, 1 addition).
  \item The distance is inverted to enable lock-free maximum selection:
        $\hat{s}_i = 131{,}072 - d^2_i$.
  \item The score and index are packed into a single \texttt{uint64}:
\end{enumerate}

\begin{equation}
  \text{packed}_i = \bigl(\hat{s}_i \ll 32\bigr) \;\big|\; i
  \label{eq:packed}
\end{equation}

The winner is determined by a single \texttt{AtomicMax} operation per thread.
Because the inverted distance occupies the high 32 bits, lexicographic
integer comparison correctly preserves distance ordering.

\paragraph{Computational Cost.}
Each thread executes exactly 5 integer operations (2 subtracts, 2 multiplies,
1 add) plus one atomic compare-and-swap.  The entire scan is memory-bound
at 4 bytes per candidate (one \texttt{GFRotation} = 2 $\times$
\texttt{uint16}).

\subsection{Multi-Backend Dispatch}
\label{sec:multi_backend}

The kernel dispatch layer supports Metal (Apple Silicon), CUDA (NVIDIA),
and CPU (fallback) backends with automatic selection via the
\texttt{SIX\_BACKEND} environment variable:

\begin{equation}
  i^* = \operatorname*{arg\,min}_{i \in \text{Nodes}}\;
        d^2(\mathbf{q}, \mathbf{n}_i)
  \label{eq:dispatch}
\end{equation}

For CUDA, multi-GPU orchestration distributes the node array across devices
with per-device base offsets, merging local winners via the packed score
representation.

\subsection{Composition Instead of Accumulation}
\label{sec:composition}

A key distinction from popcount-based retrieval (BestFill) is that the
$\operatorname{GF}(257)$ resolver operates on \emph{composed} rotational
states rather than accumulated bit densities.  When the Sequencer detects
a topological boundary event, it emits a rotation:

\begin{itemize}[nosep]
  \item \textbf{Density Spike} $\to$ $X_{90}$: $(1, 1)$
  \item \textbf{Phase Inversion} $\to$ $Y_{90}$: $(g, 0)$
  \item \textbf{Density Trough} $\to$ $Z_{90}$: $(g, 1)$
  \item \textbf{Low-Variance Flux} $\to$ $X_{180}$: $(1, 2)$
\end{itemize}

These compose via the group operation:
\begin{equation}
  (a_2, b_2) \circ (a_1, b_1) = \bigl((a_2 \cdot a_1) \bmod 257,\;\;
                                        (a_2 \cdot b_1 + b_2) \bmod 257\bigr)
  \label{eq:composition}
\end{equation}

Because composition is non-commutative, the resolver distinguishes sequences
that traverse different rotational paths even if they contain identical
content---encoding syntax as geometry.
