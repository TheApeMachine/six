\subsection{Machine Architecture}
\label{sec:vm_machine}

The Machine is the top-level entrypoint to the architecture.  It orchestrates
data loading, PrimeField construction, and the prompt-to-generation loop:

\begin{enumerate}[nosep]
  \item \textbf{Loading:}  Raw byte streams are tokenized via the Universal
        Tokenizer, sequenced by the topological Sequencer, and inserted into
        the PrimeField with their structural events driving manifold rotations.
  \item \textbf{EigenMode Calibration:}  After ingestion, a multi-scale
        co-occurrence matrix is built from the stored values and factored
        into toroidal eigenvectors---establishing the navigation phase map.
  \item \textbf{Prompt Processing:}  Input prompt bytes are converted to
        values, accumulated into a query manifold via topological event
        application, and used to seed the generation context.
\end{enumerate}

\subsection{The Generation Loop}
\label{sec:generation_loop}

Generation proceeds as follows (up to \texttt{maxGenerationSteps = 256}):

\begin{enumerate}[nosep]
  \item Derive a \emph{slotMask} by comparing the query context against
        the ExpectedReality manifold, identifying which blocks contain
        observed data and which represent structural vacuums.
  \item Execute BestFill across the PrimeField with multi-hop reasoning:
        up to \texttt{maxReasoningHops = 3} iterative searches, each time
        masking the previously matched manifold to encourage exploration.
  \item Integrate the matched span's values into the context manifold via
        the slotMask, only filling blocks that were identified as missing.
  \item Decode the best-matching manifold's readout bytes and emit them.
  \item Check for geometric closure via EigenMode phase convergence, or
        detect manifold collapse (score below threshold).
\end{enumerate}

\subsection{Multi-Hop Reasoning via Iterative Masking}
\label{sec:multi_hop}

For reasoning tasks requiring information from multiple memory locations, the
Machine supports multi-hop retrieval.  After each BestFill match, the matched
manifold is temporarily masked (\Cref{sec:masking}) and the search is
repeated.  This forces the system to recruit increasingly distant structural
neighbors, enabling transitive inference chains.
